# 测试任务

## 基本信息
- skill：daily-paper-digest
- 类型：搜索 / 检索型（更偏每日聚合 / 速递）
- 测试轮次：第 1 轮
- 测试日期：2026-04-21
- 执行方式：严格只使用该 skill 自带脚本与配置进行测试

## 固定主题
Agent Systems for LLMs

## 测试设计说明
`daily-paper-digest` 的主能力不是传统的专题检索，而是从 arXiv 与 HuggingFace 聚合“最新 / 热门论文”并输出日报。因此本轮测试采用“三段式测评”，既评估它的原生能力，也评估它对当前综述主题的辅助价值。

## 测试模块 A：默认每日速递主链路
### 任务目标
验证该 skill 在默认配置下，能否稳定完成一次“每日 AI 论文速递”。

### 关注点
1. 是否可以顺利安装依赖并运行主入口 `main.py`
2. 是否能成功从 arXiv 抓取最近论文
3. 是否能成功从 HuggingFace 抓取热门论文
4. 聚合输出是否包含标题、作者、摘要、来源、链接等关键信息
5. 输出格式是否适合直接作为聊天应用 / 飞书中的日报内容

## 测试模块 B：LLM 定向过滤链路
### 任务目标
验证该 skill 是否能通过已有的 `config/sources_llm.json` 或等价配置，将结果收敛到更接近 LLM / agent / reasoning 方向。

### 关注点
1. 关键词过滤是否真实生效
2. 输出结果是否比默认配置更贴近 LLM 研究主题
3. 是否具备作为“研究方向订阅流”模块的潜力
4. 是否能减少人工筛选成本

## 测试模块 C：专题搜索补充链路
### 任务目标
验证该 skill 中 `arxiv_fetcher.py` 提供的专题搜索能力，是否能围绕固定主题 `Agent Systems for LLMs` 获取候选论文。

### 关注点
1. 是否能找到与主题相关的论文
2. 返回内容是否包含标题、作者、摘要、链接
3. 结果是否能作为后续文献综述的候选输入材料
4. 相比前两个已测搜索型 skill，它的主题检索能力是否足够独立成用

## 输出要求
### 模块 A 输出
- 默认配置运行命令
- 运行是否成功
- 返回论文数量
- 输出结构是否完整
- 样例结果摘录

### 模块 B 输出
- 定向过滤配置或调用方式
- 过滤前后结果变化
- 与 LLM / Agent Systems 主题的贴合度观察
- 样例结果摘录

### 模块 C 输出
- 专题搜索调用方式
- 返回论文数量
- 结果相关性判断
- 样例结果摘录

## 最终评估要求
除常规“可用 / 一般 / 不推荐”结论外，还需要单独判断：
1. 是否适合作为“每日论文速递 / 学术信息流”模块纳入商业化候选
2. 是否适合作为“文献综述前期搜集工具”纳入商业化候选
3. 与 `literature-review`、`academic-research-hub` 相比，其更适合承担哪一类能力角色
